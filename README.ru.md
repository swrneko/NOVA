## 📖 О проекте

**NOVA** — это программный комплекс клиент-серверной архитектуры, представляющий собой интеллектуального голосового ассистента. Система совмещает локальные нейросети распознавания и синтеза речи с мощной облачной языковой моделью через OpenAI-совместимый API от **io.net Intelligence**. 

Уникальность ассистента заключается в глубокой интеграции с **умным домом (MQTT/Home Assistant)**, проработанном аниме-характере (персонаж Нанасаки Аи из Amagami SS) и полной оптимизации под запуск на современных ноутбуках (включая процессоры AMD Ryzen 7 без дискретных видеокарт NVIDIA) за счет применения квантования `int8` и CPU-оптимизированных ИИ-библиотек.

## 🛠️ Стек технологий

* **Язык разработки:** Python 3.10+
* **Фронтенд:** Textual TUI (Terminal User Interface с поддержкой реактивных событий, CSS-компоновки и виджетов).
* **Бэкенд:** FastAPI + Uvicorn (высокопроизводительный асинхронный сервер).
* **База данных:** SQLite + SQLAlchemy ORM.
* **Сетевой стек:** REST API (HTTP-запросы), WebSockets (двунаправленный real-time стриминг аудио и текста), MQTT (протокол обмена IoT).
* **Аудио-подсистема:** PyAudio / SoundDevice (захват звука) + Librosa (высокоточное пересэмплирование аудио с 44.1 кГц до 16 кГц / 22.05 кГц).
* **ИИ-Модели:**
  * **VAD (Voice Activity Detection):** `silero-vad` — для умного определения начала и конца речи (исключает ложные срабатывания на шум).
  * **STT (Speech-to-Text):** `faster-whisper` (`large-v3-turbo`) — оптимизированный движок Whisper на C++ с квантованием `int8`.
  * **LLM (Large Language Model):** `gpt-oss-20b` через облачный API-шлюз **io.net Intelligence** (OpenAI-совместимый контракт).
  * **TTS (Text-to-Speech):** `piper-tts` (`ru_RU-irina-medium`) — сверхбыстрый локальный синтезатор речи на базе ONNX-runtime.

## 🔒 Безопасность и Двухфакторная Аутентификация (2FA)

Проект использует многоуровневую систему защиты данных:

1. **Хэширование паролей:** Никаких паролей в открытом виде. Хранение осуществляется с применением алгоритма **bcrypt** с адаптивным соляным раундом.
2. **2FA по стандарту RFC 6238 (TOTP):** При регистрации генерируется уникальный криптографический ключ и выводится в клиент в виде **ASCII QR-кода**. Пользователь сканирует этот код любым аутентификатором (Aegis, Google Authenticator) на мобильном телефоне.
3. **Двухэтапная авторизация:**
   * Шаг 1: Проверка связки Логин/Пароль. Если они верны, сервер сообщает о необходимости 2FA.
   * Шаг 2: Клиент запрашивает 6-значный одноразовый код. При успешной проверке сервер генерирует криптографически подписанный **JWT-токен** со сроком действия 7 дней.
4. **Безопасное соединение:** Все WebSocket-запросы авторизуются с помощью полученного JWT-токена в заголовках.

## 🧠 Оптимизация ИИ под обычные ноутбуки (AMD Ryzen / Intel)

Тяжелые нейросети требуют огромных вычислительных мощностей. Чтобы проект летал на ноутбуке **AMD Ryzen 7 7840HS** без дискретной видеокарты NVIDIA, были предприняты следующие оптимизации:

* **Исключение CUDA:** Из зависимостей удалена тяжелая сборка PyTorch с CUDA. Используется CPU-оптимизированный PyTorch.
* **Квантование Faster-Whisper:** Для распознавания речи модель Whisper запущена с параметрами `device="cpu"` и `compute_type="int8"`. Это снизило потребление ОЗУ до ~1.5 ГБ и сократило время декодирования речи в 4-5 раз.
* **Локальный Piper TTS:** Piper написан на C++ и использует движок ONNX Runtime, что позволяет ему синтезировать аудио быстрее реального времени (RTF < 0.3) прямо на процессоре.
* **Гибридное облако (io.net LLM):** Локальный запуск модели 20B+ параметров требует 24ГБ+ VRAM. Использование OpenAI-совместимого API от **io.net** переносит самые ресурсоемкие вычисления на глобальные графические кластеры, сохраняя низкую задержку ответа (стриминг токенов) и разгружая процессор ноутбука.

## 🔌 Интеграция с Умным Домом (IoT / MQTT)

NOVA умеет выполнять системные команды и управлять физическим железом с помощью встроенного **MQTT-клиента**.
В `config.py` задается адрес локального брокера Home Assistant.

### Доступные голосовые функции (Tools / Function Calling):
* **Узнать время:** Ассистент запрашивает системное время, переводит его в текстовый вид без цифр (согласно правилам аниме-речи) и проговаривает вслух.
* **Управление настольной лампой (`turnOnTableLight`):** Отправляет JSON-payload в топик `nanasaki/desk-light`.
* **Управление основным светом (`turnBigLight`):** Отправляет JSON-payload в топик `nanasaki/big-light`.

## 🚀 Инструкция по установке и запуску

Для развертывания проекта рекомендуется использовать дистрибутив Arch Linux и менеджер окружений **Conda**.

### 1. Подготовка окружения
Создайте изолированное окружение с Python 3.10:
```bash
conda create -n NOVA python=3.10
conda activate NOVA
```

### 2. Установка зависимостей
Установите системные зависимости (для сборки аудио-библиотек может потребоваться `portaudio`):
* **Arch Linux:** `sudo pacman -S portaudio alsa-lib`
* **Ubuntu/Debian:** `sudo apt install portaudio19-dev libasound2-dev`

Установите Python-библиотеки:
```bash
pip install -r requirements.txt
# Установка CPU-версии PyTorch и Torchaudio (чтобы избежать CUDA-зависимостей)
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cpu
```

### 3. Загрузка моделей синтеза речи (TTS)
Создайте директорию `piper-models/` в корне проекта и скачайте туда ONNX-модель и конфигурационный файл русской озвучки "Ирина" (medium):
```bash
mkdir -p piper-models
cd piper-models
wget https://huggingface.co/rhasspy/piper-voices/resolve/main/ru/ru_RU/irina/medium/ru_RU-irina-medium.onnx
wget https://huggingface.co/rhasspy/piper-voices/resolve/main/ru/ru_RU/irina/medium/ru_RU-irina-medium.onnx.json
cd ..
```

### 4. Конфигурация
Отредактируйте параметры подключения к MQTT, ключи API и системные пути в файле `config.py`. По умолчанию пути настроены относительно домашней папки проекта.

## 🏁 Запуск проекта

Запуск системы производится в два этапа (в разных терминалах):

### Этап 1: Запуск Бэкенд-Сервера
Активируйте окружение и запустите FastAPI веб-сервер через uvicorn:
```bash
conda activate NOVA
python -m uvicorn backend.server:app --host 0.0.0.0 --port 8000
```
*Сервер инициализирует ИИ-модели (VAD, STT, TTS) и откроет порт 8000 для подключений.*

### Этап 2: Запуск Клиентского Интерфейса (TUI)
В новом окне терминала запустите клиент:
```bash
conda activate NOVA
python -m client.tui
```

## 👤 Инструкция пользователя в TUI

1. **Вкладка "Accounts 👤"**:
   * Введите желаемый логин и пароль, нажмите **Register**.
   * В терминале/клиенте появится текстовый QR-код. Отсканируйте его в приложении **Aegis** или **Google Authenticator**.
   * Введите логин и пароль снова, нажмите **Login**.
   * Система попросит 2FA-код. Введите 6 цифр из приложения на телефоне и нажмите **Verify 2FA**.
   * Статус сменится на "Authenticated". Получен авторизационный JWT-токен.

2. **Вкладка "Audio Settings 🎙️"**:
   * Выберите нужное устройство микрофона (вход) и динамика (выход) из выпадающих списков.
   * Нажмите **Apply Audio Settings**.

3. **Вкладка "Chat 🌌"**:
   * Нажмите клавишу **[ПРОБЕЛ]** (SPACE) — начнется фоновая запись микрофона.
   * Задайте вопрос голосом (например: *"Аи, сколько сейчас времени?"* или *"Включи свет на столе"*).
   * Нажмите **[ПРОБЕЛ]** (SPACE) повторно для отправки голосового сообщения.
   * Бэкенд распознает ваш голос, отправит его в LLM, получит ответ, озвучит его и пришлет аудио-поток прямо в ваши динамики, одновременно выводя переписку в чат.
